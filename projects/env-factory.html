env_factory: Environment Configuration Package

A flexible Python package for retrieving environment variables from multiple sources including local environment and AWS Secrets Manager with robust error handling and type safety.

üì¶ Installation & Quick Start

Install from PyPI

pip install env-factory

Create .env File (MANDATORY)

# .env file - Place in your project root API_KEY=my-secret-api-key DATABASE_URL=postgresql://localhost:5432/myapp DEBUG=true # For AWS functionality (REQUIRED) AWS_ACCOUNT_ID=123456789012

Load Environment Variables

from dotenv import load_dotenv load_dotenv() # Load variables from .env file # Now you can use env_factory from env_factory import get_env_variables

üí° Usage Examples

Basic Local Environment Variables

from env_factory import get_env_variables # This will read from your .env file config = get_env_variables(['API_KEY', 'DATABASE_URL']) print(config) # Output: {'API_KEY': 'my-secret-api-key', 'DATABASE_URL': 'postgresql://localhost:5432/myapp'} # Check if variables exist api_key = config.get('API_KEY') if api_key: print(f"API Key found: {api_key}") else: print("API Key not found!")

AWS Secrets Manager Integration

from env_factory import get_env_variables # Make sure AWS_ACCOUNT_ID is in your .env file! secrets = get_env_variables( keys=['DB_PASSWORD', 'JWT_SECRET'], source='aws', secret_name='my-app-secrets', region='us-east-1', role_name='my-app-role' ) print(f"Database password: {secrets.get('DB_PASSWORD')}")

Real-World Web Application Setup

from env_factory import get_env_variables # Load all your app configuration at startup app_config = get_env_variables([ 'API_KEY', 'DATABASE_URL', 'DEBUG', 'PORT' ]) # Use the configuration api_key = app_config['API_KEY'] database_url = app_config['DATABASE_URL'] debug_mode = app_config.get('DEBUG', 'false').lower() == 'true' port = int(app_config.get('PORT', '8000')) print(f"Starting app on port {port}, debug mode: {debug_mode}")

Factory Pattern Usage

from env_factory import EnvFactory # Create retrievers using factory local_retriever = EnvFactory.create_local_retriever() aws_retriever = EnvFactory.create_aws_retriever( secret_name='my-secrets', region='eu-west-1', role_name='secrets-access-role' ) # Use the retrievers config = local_retriever.get_env_variables(['PORT', 'HOST']) secrets = aws_retriever.get_env_variables(['API_TOKEN', 'WEBHOOK_SECRET'])

Multi-Environment Configuration

import os from env_factory import get_env_variables # Determine environment environment = os.getenv('ENVIRONMENT', 'development') if environment == 'production': # Use AWS Secrets Manager in production config = get_env_variables( keys=['DATABASE_PASSWORD', 'API_KEY', 'JWT_SECRET'], source='aws', secret_name='prod-app-secrets', region='us-east-1', role_name='prod-secrets-role' ) else: # Use local environment variables in development config = get_env_variables([ 'DATABASE_PASSWORD', 'API_KEY', 'JWT_SECRET' ]) print(f"Loaded {len([v for v in config.values() if v is not None])} variables")

Error Handling Best Practices

from env_factory import get_env_variables, ConfigurationError, SecretRetrievalError try: variables = get_env_variables( keys=['API_KEY', 'SECRET_TOKEN'], source='aws', secret_name='app-secrets', region='us-east-1', role_name='secrets-role' ) except ConfigurationError as e: print(f"Configuration error: {e}") except SecretRetrievalError as e: print(f"Failed to retrieve secrets: {e}") # Handle missing variables gracefully api_key = variables.get('API_KEY') if not api_key: print("‚ö†Ô∏è Warning: API_KEY not found") print("Please add it to your .env file: API_KEY=your-key-here")

üîß Technical Architecture

Factory Pattern Design

Implements the factory pattern for flexible creation of environment retrievers, allowing easy switching between local and AWS sources based on deployment environment.

Type Safety & Validation

Full type hints throughout the codebase with robust validation and error handling. Provides meaningful error messages for debugging configuration issues.

Smart Key Matching

Implements case-insensitive key matching with fallback strategies, ensuring configuration works even with slight key name variations between environments.

Extensible Architecture

Abstract base class design allows for custom environment retrieval implementations while maintaining consistent interface across different sources.

üìö API Reference

Core Functions

# Main convenience function get_env_variables( keys: List[str], source: str = "local", # 'local' or 'aws' **kwargs # Additional AWS parameters ) -> Dict[str, Optional[str]] # Factory methods EnvFactory.create_local_retriever() -> LocalEnvRetrieval EnvFactory.create_aws_retriever( secret_name: str, region: str, role_name: str ) -> AWSEnvRetrieval

Exception Classes

# Base exception for the package class EnvFactoryError(Exception): """Base exception for env_factory package""" # Configuration-related errors class ConfigurationError(EnvFactoryError): """Raised when configuration parameters are invalid""" # Secret retrieval errors class SecretRetrievalError(EnvFactoryError): """Raised when secret retrieval from AWS fails"""

üìã Best Practices

Development Setup

  • Always use .env files in project root (mandatory)
  • Load .env files with python-dotenv at startup
  • Include AWS_ACCOUNT_ID for AWS functionality
  • Add .env files to .gitignore for security

Production Deployment

  • Use AWS Secrets Manager for sensitive data
  • Implement proper IAM roles and permissions
  • Handle ConfigurationError and SecretRetrievalError
  • Validate required variables are present

Code Organization

  • Load configuration at application startup
  • Use environment-specific source selection
  • Implement graceful fallbacks for missing keys
  • Enable debug logging for troubleshooting
