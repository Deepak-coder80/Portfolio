pg-migrate: PostgreSQL Migration CLI

A simple, reliable CLI tool for migrating PostgreSQL databases between different hosts with full schema and data preservation.

üöÄ Why pg-migrate?

Solving real-world database migration challenges that developers and DevOps engineers face daily:

Development to Production Migrations

  • Migrate local development databases to staging/production servers
  • Handle complex cloud provider migrations (AWS RDS, Google Cloud SQL, Azure)
  • Support different environments with varying SSL requirements

Database Cloning & Backup Restoration

  • Create exact copies of production databases for testing
  • Migrate databases between different cloud providers
  • Handle disaster recovery scenarios requiring cross-region migrations

Environment Synchronization

  • Keep development, staging, and production databases in sync
  • Share database snapshots with team members
  • Migrate databases for CI/CD pipeline testing

üí° Real-World Examples

Development to Cloud Migration

# Migrate local dev database to AWS RDS pg-migrate \ --source-db myapp_dev \ --source-host localhost \ --source-user postgres \ --dest-host myapp.abc123.us-east-1.rds.amazonaws.com \ --dest-user postgres \ --dest-ssl require

Cross-Cloud Migration

# Migrate from AWS RDS to Google Cloud SQL pg-migrate \ --source-db production \ --source-host aws-prod.abc123.us-east-1.rds.amazonaws.com \ --source-user dbadmin \ --source-ssl require \ --dest-host gcp-prod.abc123.us-central1.sql.goog \ --dest-user postgres \ --dest-ssl verify-ca

Database Cloning

# Clone production to staging environment pg-migrate \ --source-db myapp_prod \ --source-host prod.example.com \ --source-user readonly_user \ --dest-host staging.example.com \ --dest-db myapp_staging \ --dest-user postgres

Built with Go

Cloud Ready
