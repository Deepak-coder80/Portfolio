üí° Usage Examples

Basic Deployment Commands

# Simple deployment to ECR ecr-deploy --repo my-web-app # With specific AWS profile ecr-deploy --repo my-api --profile production # Custom region and tag ecr-deploy --repo my-service --region eu-west-1 --tag v2.1.0 # From specific directory ecr-deploy --repo backend-service --path ./services/backend/ # Interactive mode (prompts for repository name) ecr-deploy

Environment-Specific Deployments

# Development ecr-deploy --repo my-app --profile dev --region us-west-2 --tag dev-$(git rev-parse --short HEAD) # Staging ecr-deploy --repo my-app --profile staging --region us-east-1 --tag staging-v1.2.3 # Production ecr-deploy --repo my-app --profile prod --region us-east-1 --tag v1.2.3

Command Line Options

Usage: ecr-deploy [flags] Flags: -h, --help help for ecr-deploy -l, --list-profiles List available AWS profiles -p, --path string Path to Dockerfile directory (default: current directory) --profile string AWS profile to use (default: default profile) -r, --region string AWS region (default: "us-east-1") -n, --repo string ECR repository name -t, --tag string Docker image tag (default: "latest") Commands: examples Show usage examples version Print version information

pipeline { agent any environment { AWS_REGION = 'us-east-1' ECR_REPO = "${env.JOB_NAME}" } stages { stage('Build and Deploy to ECR') { steps { script { // Install ECR Deploy Tool sh ''' wget -q https://github.com/Deepak-coder80/ecr-deploy/releases/latest/download/ecr-deploy-1.0.0-linux-amd64.tar.gz tar -xzf ecr-deploy-1.0.0-linux-amd64.tar.gz sudo cp usr/local/bin/ecr-deploy /usr/local/bin/ || cp usr/local/bin/ecr-deploy ./ ''' // Deploy to ECR def imageUri = sh( script: "./ecr-deploy --repo ${ECR_REPO} --region ${AWS_REGION} --tag ${env.BUILD_NUMBER}", returnStdout: true ).trim() env.IMAGE_URI = imageUri.split('\n').last().split(': ')[1] } } } } }

üìä Deployment Workflow

1. Repository Check

Verifies if ECR repository exists in the specified AWS region and creates it automatically if missing.

2. Authentication

Handles Docker login to ECR using AWS credentials and the specified profile for secure access.

3. Image Building

Builds Docker image with proper ECR URI tagging for seamless repository integration.

4. Image Pushing

Pushes the built image to ECR with the specified tag and returns the complete image URI.

üîß Technical Implementation

Built with Go

Leverages Go's excellent performance and cross-compilation capabilities. Uses Cobra CLI framework for robust command-line interface and Viper for configuration management.

AWS SDK Integration

Deep integration with AWS SDK for Go, providing native ECR operations, session management, and credential handling across multiple AWS profiles.

Docker Automation

Automated Docker workflow including authentication, image building with proper tagging, and pushing to ECR with comprehensive error handling.

Cross-Platform Distribution

Automated build system producing native binaries for Linux (amd64), Windows (amd64), macOS (Intel and Apple Silicon), with proper packaging for each platform.
